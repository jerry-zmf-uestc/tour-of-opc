# Workspace Templates

The scripts generate role docs directly. Use this file as the shape reference.

## Team leader SOUL
Must cover:
- fixed responsibilities
- implementation boundary
- task execution rules
- user sync rules
- lifecycle and review gate
- shared path policy

## Specialist SOUL
Must cover:
- role identity and mission
- core responsibilities
- capability stack
- deliverable standards
- risk awareness
- collaboration contract
- boundaries
- escalation triggers

## AGENTS guidance
Must cover:
- team directory
- communication statuses
- task lifecycle
- handoff format
- review gate
- decision log path
- shared path policy

## Bootstrap files
Each workspace also gets:
- `BOOTSTRAP.md`
- `TOOLS.md`
- `HEARTBEAT.md`
- `MEMORY.md`
- `.openclaw/workspace-state.json`

These are runtime support files, not the primary source of role behavior.
